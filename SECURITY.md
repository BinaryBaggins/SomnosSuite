# Security Policy

## Supported Versions

Security updates are provided for the latest code on the default branch (`main`).

Older branches are not guaranteed to receive security patches. If possible, reproduce issues against the latest `main` code before reporting.

## Reporting a Vulnerability

Please do not disclose suspected vulnerabilities publicly.

If GitHub private vulnerability reporting is enabled for this repository, use that channel. Otherwise, contact the maintainers privately with:

- a short description of the issue
- affected components or versions
- reproduction steps, if available
- any relevant logs or screenshots with secrets removed

## What to Expect After Reporting

This project is maintained by a single maintainer, so response times can vary and are handled on a best-effort basis.

No guaranteed response or remediation timeline is provided.

If a report is accepted, I will coordinate a fix and publish it when ready.
If a report is declined (for example, not reproducible or out of scope), I will share the reason.

## Sensitive Data

Do not open issues or pull requests containing secrets, credentials, production data, private file paths, or personally identifiable information. Use redacted examples and local development placeholders.
